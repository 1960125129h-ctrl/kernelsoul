这是什么

Kernelsoul 是一个纯 Python 的 AI 角色扮演后端引擎。不同于大多数"拼接 Prompt 调 API"的方案，它将角色扮演拆解为多个独立子系统——状态机、解析器、记忆管理器、DSL 编译器、世界书检索、进化触发器、插件管理器——每个可单独测试、替换、扩展。


自 v1.0 起迭代 12 个版本，v2.4 覆盖 21 章开发手册，代码量 ~6500 行，支持 DeepSeek / OpenAI 兼容 API，通过 FastAPI + SSE 暴露 REST 接口。

二、核心模块清单

GameState & StateManager

结构化游戏状态：HP/EP/Emotion/Items/Goodwill 等。双序列化机制（JSON + state.sav），支持跨会话恢复。

MemoryManager

三层压缩架构：轻量(10轮200字)→深度(30轮2000字)→史诗(200轮)。Gram矩阵解耦压缩提示，自动上下文裁剪。

ParserEngine

三级解析：JSON代码块提取→自然语言模式匹配→回退。支持HP/EP/情感/物品/金币/场景等字段的自动提取。

DSL Compiler + RuleCompiler

声明式角色行为DSL：@trigger条件→优先级排序→执行动作。支持 say/do/set_state/add_item 等多种动作类型。

WorldBookRetriever

关键词+正则+二级键三维匹配检索，加权排序（精确>正则>子串>二级），动态上下文注入。

EvolutionTrigger

规则随对话进程动态演化：状态阈值触发新规则加载，全局规则+角色规则双层管理。

CharacterCardLoader

V2(V1 PNG/JSON单文件)→V3(V2扩展)→Kernelsoul(文件夹格式)三级格式兼容，自动转换。

PluginManager

钩子系统：on_user_input / on_ai_response / filter。插件自动发现、异常隔离、生命周期管理。

AIBridge

OpenAI兼容客户端，支持流式+非流式生成，可切换 DeepSeek / OpenAI / 本地模型。

Context & ContextWrappers

系统提示+用户提示组装：角色描述→状态文本→世界书→压缩摘要→对话历史→规则编译→用户输入。

PathResolver

文件系统路径管理：角色目录/会话目录/存档/日志/草稿/记忆的路径绑定与创建。

SessionMeta

会话元数据：总轮数、时间戳、角色绑定。支持序列化持久化。

DraftManager

草稿保存与恢复：角色草稿/会话草稿，支持时间戳版本管理。

CoreEngine (V4Engine)

总装模块：初始化所有子系统，提供 build_prompt() + switch_character() + switch_session() 等顶层API。

API Server

FastAPI 后端：9个REST端点 + SSE流式对话。角色列表/切换/状态查询/会话管理/聊天/渲染模式。

语义渲染引擎 v2.4

第二十一章新子系统：语义轴→子空间投影→Warp标定→SLERP更新→KNN短语检索→ContextPacket。三模式(direct/semantic/hybrid)可切换。语义引擎+渲染管线+语料库3模块1004行。

总结

Kernelsoul 是一个从 v1.0 到 v2.4 持续迭代 12 个版本的模块化 AI 角色扮演引擎，覆盖 21 章手册规范、20 个核心模块、~6500 行 Python 代码。


核心设计理念是将角色扮演拆解为独立子系统——状态机、解析器、三阶记忆、DSL编译器、世界书、进化触发、插件钩子——每个可独立测试替换。v2.4 在此基础上新增语义渲染引擎（第二十一章），引入向量空间投影、Warp非线性映射、SLERP状态更新、KNN短语检索，提供 direct / semantic / hybrid 三种渲染模式。


当前状态：所有子系统完成并测试通过（35/35），已打包 MIT 开源版。核心瓶颈为 embedding API 依赖（DeepSeek不支持），接入后可激活语义引擎完整能力。未来方向包括 gRPC API、Unity/Unreal SDK 和 pip 安装包。
