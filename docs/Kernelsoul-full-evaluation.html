一句话定位

Kernelsoul 是面向 AI 角色扮演的确定性角色引擎——在 LLM 与角色卡之间插入持久化状态层、规则执行层和语义渲染管线，让角色卡从"被阅读的故事"升级为"被执行的程序"。

三、Kernelsoul 核心竞争优势

🎯 唯一确定性角色引擎

所有竞品都是"前端对话界面"，Kernelsoul 是唯一自带强类型状态机 + 确定性规则执行器的项目。GameState 的 int/bool/list 白名单机制让角色行为不再是概率性 prompt 暗示，而是回合级强制执行的规则。

🧠 语义渲染管线（行业首创）

语义叙事渲染引擎（第21章）通过语义轴、Warp函数、Gram矩阵解耦读数，将 GameState 数值映射为叙事描述性短语。这在所有 AI RP 平台中是 首创功能——竞品只有"数值直出"一种模式。

📜 SLL 剧本语言

Kernelsoul 定义了Kernelsoul Logic Language（SLL），一个专门面向角色行为规则的 DSL。递归下降解析器直接将其编译为可执行规则，不需要 LLM 在每次对话时重新理解角色行为边界。

🧩 决策溯源能力

唯一支持"WHY"关键字查询决策原因的系统。用户可以查看角色某次行为触发的具体规则、状态变化链路。这在 LLM 黑盒输出时代提供了关键的可解释性。

🎲 草稿与版本管理

多草稿生成 + 收藏 + 版本回溯。一套完整的对话分支管理能力，竞品无一提供。

💪 三层记忆压缩

轻量(10轮/200字) → 深度(50轮/2500字) → 史诗(200轮/5000字) 三级分层架构，由具体事件频次驱动压缩触发，而非简单的 Token 计数截断。上下文利用率远超所有竞品。

四、市场定位与竞品格局

👑 SillTavern（主导者）

  • 约 80% 用户市场
  • 300+ 扩展的成熟生态
  • chara_card_v2 事实标准
  • 优点：UI 完善、社区大
  • 局限：纯前端，无状态/规则

🥈 RisuAI / Agnai（挑战者）

  • 各占 5-10% 用户
  • 部分 UI 特色功能
  • 同样纯前端无状态
  • RisuAI 有 regex 脚本
  • 生态远不如 ST

⚡ Kernelsoul（颠覆者）

  • 不是前端，是引擎
  • 不替代 ST，增强其能力
  • 插件模式安装到 ST
  • 独立运行 + SDK
  • 无竞品生态依赖

七、当前限制与待改进

⚠️ 嵌入层待接入

语义引擎需要外部 embedding 模型（OpenAI text-embedding-3-small / BGE-Large-ZH），目前用 mock 函数占位。接入后可激活完整的语义空间。

⚠️ 前端缺失

CLI 模式已可用，但 gradio 前端已被放弃。需对接第三方前端（SillyTavern 插件模式是主要方案）或重建轻量 UI。

🔜 生态建设

插件市场、gRPC API、Unity/Unreal SDK 在手册中规划但未实现。社区和生态是 Kernelsoul 能否从技术项目走向流行平台的关键。

🔜 文档完备化

手册内容全面但偏实现细节，缺少面向非开发者的快速入门和角色创作者指南。

八、综合结论

Kernelsoul 在 AI 角色扮演领域拥有独一档的技术差异——它是唯一将游戏状态机、确定性规则引擎和语义叙事渲染管线和专业剧本 DSL 集成到单一引擎中的项目。
