新子系统详解

21.2-21.3 数学框架

语义轴Dᵢ = embed(正面锚) − embed(负面锚)，每个游戏维度一条轴
状态点z = r·p — 方向向量p + 强度标量r，在高维球面上移动
Warp函数数值Δv通过阶段密度曲线映射为语义Δt，临界区敏感性可调
Gram矩阵G = AᵀA 解耦坐标轴，τ(e) = G⁻¹Aᵀ(e−c) 投影到轴空间
子空间SVD降维到k=20维，忽略与游戏状态无关的嵌入维度
SLERP大角度用球面线性插值保证路径在球面上，小角度用余弦近似

21.4-21.5 渲染管线与语料库

事实标签语料库中每条短语预计算在各轴上的坐标τ(s)，4条件冲突检测
KNN检索加权多因子排序：d_fact(事实准确)+d_style(风格)+d_momentum(动量)+d_history(去重)
LLM降级KNN无结果时组装ContextPacket→LLM生成新短语→准入审计
Miss Buffer记录检索失败的状态，DBSCAN聚类检测语料缺口，触发短语生成
准入审计5项校验：方向·覆盖·可逆性·冲突·风格，新短语需通过才能入库

21.6 集成与向后兼容

渲染模式direct（v2.3行为，默认）· semantic（全量语义）· hybrid（高权重维度语义+低权重数值）
角色配置character_semantic.json — 定义每维度的正/负锚点文本、Warp阶段描述、设计权重
APIPOST /api/render_mode {"mode":"semantic"} / GET /api/render_mode
位置插入在规则执行之后、Prompt构造之前，direct模式下完全透传

总结

v2.4 是 v2.3.1 的增量更新：所有 17 个核心模块保持不变，新增第二十一章"语义渲染引擎"作为可选子系统，通过 /render_mode 接口在 direct/semantic/hybrid 间切换，默认 direct 模式与 v2.3.1 行为完全一致。


新增内容体量：1章手册规范 + 3个新模块（semantic_engine/renderer/phrase_library，1004行）+ 1个配置文件类型 + 2个API端点 + 17项测试。


已知限制：语义渲染的数学核心（Gram矩阵·子空间投影·KNN检索）依赖embedding API，当前因DeepSeek不支持而处于待激活状态。向量空间层激活后预计叙事质量有进一步提升空间。


向后兼容：v2.3.1 用户无需任何迁移——默认 direct 模式下行为完全不变，仅在主动调用 /api/render_mode 切换后才启用新功能。
